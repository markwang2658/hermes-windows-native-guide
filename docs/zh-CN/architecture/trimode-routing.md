## 二、四种模式详解

### 2.1 Chat 模式（默认对话）

| 属性 | 值 |
|------|-----|
| **触发条件** | 纯文本消息，无附件 |
| **默认模型** | Kimi K2.6 (云端) |
| **响应速度** | ~2 秒 |
| **成本** | 按 Token 计费 |
| **适用场景** | 日常对话、代码编写、问题解答 |

**为什么聊天走云端？**
- Kimi K2.6 是当前最强的中文对话模型之一
- 云端推理能力强，响应速度快
- 成本可控（按使用量付费）

```yaml
# cli-config.yaml 核心配置
model:
  default: "kimi-k2.6"           # 或 kimi-k2.0711-preview
  provider: "kimi-coding"         # Moonshot AI 提供商
  base_url: "https://api.moonshot.cn/v1"
```

---

### 2.2 Vision 模式（图片识别）

| 属性 | 值 |
|------|-----|
| **触发条件** | 上传图片文件 (.png, .jpg, .jpeg, .gif, .webp) |
| **默认模型** | GLM-4.6V-Flash (本地) |
| **运行环境** | LM Studio / Ollama |
| **响应速度** | 3-5 秒（取决于 GPU） |
| **成本** | 免费（本地推理） |
| **隐私** | ✅ 数据不出机器 |

**为什么图片走本地？**
- 隐私保护：截图可能包含敏感信息
- 零成本：无需为每张图片付费
- 无网络依赖：离线可用

```yaml
# cli-config.yaml 视觉辅助模型配置
auxiliary:
  vision:
    provider: "lmstudio"           # 或 "custom"
    model: "glm-4.6v-flash:q4"     # 量化版本节省显存
    base_url: "http://127.0.0.1:1234/v1"  # LM Studio 默认端口
    timeout: 30                    # 图片分析超时时间（秒）
```

---

### 2.3 Audio 模式（语音转写）

| 属性 | 值 |
|------|-----|
| **触发条件** | 上传音频文件 (.mp3, .wav, .m4a, .ogg) |
| **默认模型** | faster-whisper (本地) |
| **运行环境** | Python 本地 + GPU 加速（可选） |
| **响应速度** | 1-3 秒（短音频）/ 10-30 秒（长音频） |
| **成本** | 免费 |
| **隐私** | ✅ 语音数据不出机器 |

**支持的音频格式和语言：**

| 格式 | 支持 | 备注 |
|------|------|------|
| MP3 | ✅ | 最常用格式 |
| WAV | ✅ | 无损，但文件较大 |
| M4A | ✅ | Apple 设备录音 |
| OGG | ✅ | 开源格式 |

```yaml
# cli-config.yaml 语音转写配置
stt:
  enabled: true
  local:
    model: "base"                 # tiny/base/small/medium/large-v3/turbo
    # model: "small"              # 推荐：速度与精度的平衡点
    # language: "zh"              # 可选：强制指定语言（默认自动检测）
```

**模型选择指南：**

| 模型 | 显存需求 | 速度 | 精度 | 推荐场景 |
|------|----------|------|------|----------|
| `tiny` | ~1GB | ⚡⚡⚡ | ★☆☆ | 快速测试 |
| `base` | ~2GB | ⚡⚡ | ★★☆ | **日常推荐** |
| `small` | ~3GB | ⚡ | ★★★ | 高精度需求 |
| `medium` | ~5GB | 🐢 | ★★☆ | 不推荐（性价比低） |
| `large-v3` | ~10GB | 🐢🐢 | ★★★★ | 最高精度 |

### 2.4 Text 模式（文件处理）

| 属性 | 值 |
|------|-----|
| **触发条件** | 上传文本文件 (.md, .py, .js, .txt, .json, .yaml 等) |
| **默认模型** | GLM-4.6V-Flash (本地) |
| **运行环境** | LM Studio / Ollama |
| **响应速度** | 2-4 秒（取决于文件大小） |
| **成本** | 免费 |
| **隐私** | ✅ 代码/文档不出机器 |

**典型用途：**

| 文件类型 | 用途 | 示例 |
|----------|------|------|
| `.md` | 文档摘要、内容问答 | README 分析 |
| `.py/.js` | 代码解释、Bug 排查 | 函数逻辑说明 |
| `.json/.yaml` | 配置文件解读 | config.yaml 解释 |
| `.txt` | 日志分析、文本总结 | 错误日志排查 |

```yaml
# cli-config.yaml 文本处理通过 auxiliary.web_extract 或主模型
auxiliary:
  web_extract:
    provider: "lmstudio"           # 复用本地模型
    model: "glm-4.6v-flash:q4"
    base_url: "http://127.0.0.1:1234/v1"
```

## 四、技术决策理由

### 为什么采用混合云+本地架构？

```
传统方案（全云端）:
  用户 → 云端 API → 结果
  问题: 隐私风险、成本累积、依赖网络

传统方案（全本地）:
  用户 → 本地模型 → 结果
  问题: 需要 GPU、模型能力有限、冷启动慢

Hermes 方案（混合智能路由）:
  文字 → Kimi K2.6 (云端强模型)
  图片 → GLM-4.6V (本地隐私保护)
  语音 → Whisper (本地零成本)
  文件 → GLM-4.6V (本地安全)

  优势: 兼顾能力、成本、隐私三者
```

### Windows 特殊处理

| 问题 | 解决方案 | 说明 |
|------|----------|------|
| WinError 5 权限错误 | 自动重试机制 | 文件锁冲突时等待后重试 |
| CUDA DLL 版本不匹配 | 自动降级 | cublas64_13 → cublas64_12 |
| 路径分隔符 | 统一转换 | `/` → `\` 自动处理 |
| 长路径限制 (>260 字符) | 启用长路径支持 | `\\?\` 前缀 |

## 五、数据流详解

```
用户操作示例: 发送一张截图 + 文字描述

Step 1: WebUI 接收输入
  ├── 检测到图片附件 → 标记为 Vision 模式
  └── 检测到文字内容 → 作为上下文传递

Step 2: Agent 路由决策
  ├── auxiliary.vision.provider = "lmstudio"
  ├── 调用 http://127.0.0.1:1234/v1/chat/completions
  └── 模型: glm-4.6v-flash:q4

Step 3: 本地模型推理 (LM Studio)
  ├── 加载模型到 GPU 显存
  ├── 执行视觉理解任务
  └── 返回识别结果

Step 4: Agent 整合回复
  ├── 结合图片识别结果
  ├── 结合用户文字问题
  └── 生成最终回复（可能调用主模型润色）
```

## 六、性能调优建议

### 6.1 降低显存占用

```yaml
# 使用量化模型 (Q4/Q5 vs FP16)
model: "glm-4.6v-flash:q4"    # 4-bit 量化，更省显存
```

| 量化等级 | 显存占用 | 精度损失 | 推荐度 |
|----------|----------|----------|--------|
| FP16 (无量化) | 100% | 0% | GPU ≥ 12GB |
| Q5 | ~60% | < 2% | ⭐⭐⭐ **推荐** |
| Q4 | ~45% | 3-5% | ⭐⭐ GPU ≤ 8GB |
| Q3 | ~35% | > 10% | ❌ 不推荐 |

### 6.2 提升响应速度

| 方法 | 效果 | 适用场景 |
|------|------|----------|
| 模型常驻内存 | 冷启动从 30s → 2s | 频繁使用本地模型 |
| GPU 加速 (CUDA) | 速度提升 5-10x | 有 NVIDIA 显卡 |
| 批量处理 | 多文件并行 | 批量分析场景 |

## 七、常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 图片上传后无反应 | LM Studio 未启动或端口错误 | 检查 `http://127.0.0.1:1234` 是否可访问 |
| 语音转写报错 | faster-whisper 未安装 | `pip install faster-whisper` |
| 本地模型很慢 | 使用 CPU 推理而非 GPU | 安装 CUDA 版本的 PyTorch |
| 模型切换不生效 | 缓存问题 | 重启 Agent 或清除会话 |

> 更多故障排查请参阅 [故障排查大全](../troubleshooting/index.md)
